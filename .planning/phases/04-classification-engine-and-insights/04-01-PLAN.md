<objective>
Build the classification engine core: a weighted rule engine that classifies failing tests into 8 categories using deterministic signals, and an LLM client that handles cases where rule confidence falls below threshold.

Purpose: Deterministic-first classification means most tests are categorized instantly without API cost; LLM is the precision layer for ambiguous cases only, keeping cost predictable.
Output: src/classifier/ package with rule_engine.py, llm_client.py, cost_estimator.py, and models.py
</objective>

from pydantic import BaseModel
from typing import Literal, Optional

ClassificationCategory = Literal[
    "test_design_issue",
    "product_regression",

    full_title: str

**src/classifier/rule_engine.py** — Weighted signal accumulator:

Named constants at module top (all calibratable, per STATE.md flag):
- `RULE_CONFIDENCE_THRESHOLD = 0.8` — rule must reach this to skip LLM
- `LLM_UNCERTAIN_FLOOR = 0.6` — LLM result below this → category becomes "uncertain"
- `SIGNAL_WEIGHTS: dict[str, float]` — one key per named signal, used in Signal definitions for traceability

Define `Signal` as a `@dataclass` with fields: `name: str`, `category: str`, `weight: float`, `predicate: Callable[[CorrelatedTest, ParsedTest, Optional[ParsedSpecAnalysis]], bool]`

Define `SIGNALS: list[Signal]` covering all 8 categories — implement at minimum 2 signals per category:

| category | signal name | predicate |
|---|---|---|
| infrastructure_instability | infra_pattern_match | ct.has_infra_signal and not ct.has_auth_signal |
| infrastructure_instability | multi_infra_layer | len(ct.primary_infra_layers) >= 2 |
| auth_session_issue | auth_pattern_match | ct.has_auth_signal |
| auth_session_issue | auth_keyword | "401" in (pt.first_error_message or "") or "unauthorized" in (pt.first_error_message or "").lower() |
| cloud_provisioning_delay | async_flow | ct.has_async_signal |
| cloud_provisioning_delay | async_with_timeout | ct.exercises_async_flow and "timeout" in (pt.first_error_message or "").lower() |
| test_design_issue | fragile_locator | sa is not None and len(sa.fragile_locators) > 0 |
| test_design_issue | hard_coded_wait | sa is not None and len(sa.hard_coded_wait_ms) > 0 |
| race_condition | passed_on_retry | pt.retry_count > 0 and pt.status == "flaky" |
| race_condition | timing_keyword | "race" in (pt.first_error_message or "").lower() or "concurrent" in (pt.first_error_message or "").lower() |
| data_pollution | data_keyword | any(kw in (pt.first_error_message or "").lower() for kw in ["duplicate", "already exists", "constraint", "unique violation"]) |
| data_pollution | data_pattern_match | any(fp.category == "data_pollution" for fp in ct.failure_pattern_matches) |
| product_regression | no_infra_no_design | not ct.has_infra_signal and not ct.has_auth_signal and (sa is None or (len(sa.fragile_locators) == 0 and len(sa.hard_coded_wait_ms) == 0)) and pt.retry_count == 0 |
| product_regression | assertion_failure | "AssertionError" in (pt.first_error_message or "") or "expect(" in (pt.first_error_stack or "") |
| non_deterministic_ai | ai_keyword | any(kw in (pt.first_error_message or "").lower() for kw in ["non-deterministic", "hallucination", "ai response", "llm output"]) |
| non_deterministic_ai | ai_pattern_match | any(fp.category == "non_deterministic_ai" for fp in ct.failure_pattern_matches) |

Implement `run_rule_engine(ct, pt, sa) -> tuple[str | None, float]`:
1. Sum weights per category for all matching signals (defaultdict)
2. Compute max_possible per category (sum of all weights for signals in that category)
3. Normalise: `score / max_possible` per category, capped at 1.0
4. Find winning category (max normalised score)
5. If winning score < RULE_CONFIDENCE_THRESHOLD → return (None, winning_conf) to trigger LLM
6. Return (winning_cat, winning_conf)

Guard: if `sa is None`, all predicates referencing `sa.fragile_locators` or `sa.hard_coded_wait_ms` must check `sa is not None` first (Pitfall 6 in RESEARCH.md).

**src/classifier/llm_client.py** — LLM classification via forced tool_use:

Constants at module top:
```python
MAX_LOG_CHARS = 8_000   # 8000 chars / 4 chars-per-token ≈ 2000 tokens
HAIKU_MODEL = "claude-haiku-4-5"
SONNET_MODEL = "claude-sonnet-4-6"
```

Implement `_CLASSIFY_TOOL` dict exactly as shown in RESEARCH.md Pattern 2. The tool schema must include all five fields: `category` (enum of 8 categories + "uncertain"), `probability`, `reasoning_chain`, `fix_recommendation`, `summary_paragraph`. All five are `required`.

Implement `extract_log_context(error_message, error_stack, retry_errors) -> str`:
- Concatenate all non-None error text (message, stack, retry errors list) with newlines
- Filter to lines matching `r'\b(ERROR|WARN|FATAL|EXCEPTION|Traceback)\b'` (case-insensitive)
- If no severity lines found, fall back to full concatenated text (some test formats lack severity markers)
- Truncate to MAX_LOG_CHARS using simple slicing (character heuristic for truncation — do NOT call count_tokens here, per RESEARCH.md anti-pattern)

Implement `async classify_with_llm(test_title, log_context, use_sonnet) -> dict`:
- Select model: SONNET_MODEL if use_sonnet else HAIKU_MODEL
- Call `client.messages.create()` with `tool_choice={"type": "tool", "name": "classify_test"}`
- DO NOT enable extended thinking (`thinking={"type":"enabled"}`) — incompatible with tool_choice type=tool (Pitfall 2 in RESEARCH.md)
- Extract the tool_use block from response.content (first block with `.type == "tool_use"`)
- Apply LLM_UNCERTAIN_FLOOR: if `result["probability"] < LLM_UNCERTAIN_FLOOR`, set `result["category"] = "uncertain"`
- Return the input dict from the tool_use block plus `usage` from `response.usage`

Implement `async classify_test(ct, pt, sa) -> ClassificationResult` — full pipeline:
1. Run rule engine first: `rule_category, rule_conf = run_rule_engine(ct, pt, sa)`
2. If rule_category is not None: return ClassificationResult with method="rule_engine", tokens_used=0, reasoning_chain=[]
3. Else: build log_context via extract_log_context using `pt.first_error_message`, `pt.first_error_stack`, `[r.error_message or "" for r in pt.retries]`
4. Set `use_sonnet = ct.correlation_confidence < 1.0` (no KG match → more context needed → use sonnet)
5. Call classify_with_llm; set method="llm_haiku" or "llm_sonnet"
6. Set tokens_used = response_usage.input_tokens + response_usage.output_tokens
7. Return ClassificationResult
