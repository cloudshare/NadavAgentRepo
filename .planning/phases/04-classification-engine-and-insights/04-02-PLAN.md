Implement `generate_rule_summary(category: str, probability: float, signal_names: list[str]) -> str`:
- Returns a one-paragraph NL summary for the rule_engine classification path
- Include category, confidence as percentage, and a plain-English description of what was detected
- Use signal_names to explain why: e.g., "Matched signals: infra_pattern_match, multi_infra_layer"
- Example: "The rule engine classified this test as infrastructure_instability with 90% confidence. The following signals were matched: infra_pattern_match (a failure pattern from the CloudShare knowledge graph matched the error text) and multi_infra_layer (the test exercises 2 or more infrastructure layers). No LLM call was required."
